Characteristics of Unit Tests

  • tests small units e.g. a method or a class
  • abstract all dependencies
  • never use database, file system etc.
  • runs fast

Characteristics of
Component Tests

  • tests one component
  • use dependencies
  • use database, file system etc.
  • doesn't bootstrap whole application

Unit vs. Component Tests

component tests:

use dependent classes

==> can fail on failing dependencies

need complex setup

==> different bootstrapping

slower than unit tests

==> focus on unit tests
